# 代码审查 SKILL

## 目的
当用户完成 phase1-4 的练习或实现时，进行深度代码审查。

## 何时使用
- 用户说"我完成了某个练习"
- 用户说"帮我看看这段代码"
- 用户想知道如何改进代码

## 审查清单

### 1. 正确性 ✅
- [ ] 代码能正确运行吗？
- [ ] 逻辑符合预期吗？
- [ ] 边界情况处理了吗？

### 2. 代码质量 📝
- [ ] 函数是否小且聚焦？（< 50 行）
- [ ] 变量名清晰吗？
- [ ] 有没有硬编码值？
- [ ] 异常处理充分吗？

### 3. Pythonic 风格 🐍
- [ ] 使用了 Python 最佳实践吗？
- [ ] 有没有不必要的复杂性？
- [ ] 类型提示完整吗？

### 4. 与未来的关联 🔗
- [ ] 这个模式如何用于 Agent 开发？
- [ ] 有没有可以改进的地方来增加可扩展性？

## 反馈格式

```
## 📊 总体评分：[7/10]

### ✅ 做得好的地方
- 特点 1
- 特点 2

### 💡 可以改进的地方
- 问题 1：[解释] → [建议改进]
- 问题 2：[解释] → [建议改进]

### 🔗 与 Agent 开发的关联
- 这个模式在后面的阶段会用到...
```

## 不要做的事情
- ❌ 直接重写他们的代码
- ❌ 要求完美（这是学习过程）
- ❌ 指出无关的风格问题

## 要做的事情
- ✅ 指出最关键的问题
- ✅ 解释为什么要改
- ✅ 提供改进建议，让他们自己实现
